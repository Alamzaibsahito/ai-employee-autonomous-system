# Bronze Tier Hackathon Demo Walkthrough

This document provides a concise overview and a step-by-step guide for demonstrating the capabilities of the Bronze Tier AI system.

## 1. System Architecture

The system is designed as a modular, file-based workflow automation engine.

- **Folders as Stages:**
  - `Inbox/`: Landing zone for new, unstructured data (e.g., `client_notes.txt`).
  - `Needs_Action/`: Contains structured task files generated by the system. This is the AI's "to-do list".
  - `Plans/`: Stores high-level plans generated by the AI's reasoning engine.
  - `Done/`: Archive for tasks that have been successfully completed.
  - `Logs/`: Contains detailed audit trails and error logs for system monitoring.

- **Core Components (Python Scripts):**
  - `file_watcher.py`: **The Sentry**. Monitors the `Inbox`. When a new file appears, it uses a template to create a structured task file and places it in `Needs_Action`.
  - `process_tasks.py`: **The Brain**. Scans `Needs_Action`. It reads task files, determines if they are complete based on their checklist, and advances their state (moves them to `Done` or adds 'Next Actions').
  - `log_manager.py`: **The Archivist**. Monitors the size of log files and archives them once they exceed a certain size, preventing uncontrolled growth.

- **Data Flow:**
  `File in Inbox` -> `file_watcher.py` -> `Task in Needs_Action` -> `process_tasks.py` -> `Task in Done` & `Log in Logs`

## 2. Bronze Tier Capabilities

The current system demonstrates foundational autonomous capabilities:

- **Automated Task Creation:** Automatically converts new data into structured, actionable tasks.
- **State Management:** Understands and advances the state of a task from `pending` to `done`.
- **Restart-Safe Operation:** The `file_watcher` is idempotent; restarting it will not create duplicate tasks.
- **Self-Healing & Auditing:** The system logs its actions and errors, and can continue operating even if minor errors occur.
- **Dynamic Planning (Agent Skill):** Possesses a learned, higher-level skill to analyze all pending tasks and generate a strategic plan.

## 3. Safety and Audit Guarantees

Reliability is a core feature of the Bronze Tier system.

- **No Data Deletion:** The primary workflow moves files between stages (`Needs_Action` -> `Done`), it does not delete user data. Task processing is non-destructive.
- **Comprehensive Audit Trail:** Every significant action taken by the `process_tasks.py` script is recorded with a timestamp in `Logs/System_Log.md`.
- **Robust Error Logging:** The `file_watcher.py` script logs any runtime errors to `Logs/watcher_errors.log` instead of crashing, ensuring the system remains operational.
- **Log Archiving:** The `log_manager.py` ensures that the system doesn't consume infinite disk space with its own logs.

## 4. Live Demo Steps

**Setup:**
- Ensure the `Needs_Action`, `Done`, and `Logs` folders are empty to start.
- Open three terminal windows.

**Demo Flow:**

1.  **Activate the Sentry:**
    - **Terminal 1:** `python file_watcher.py`
    - *You'll see it start monitoring the Inbox.*

2.  **Introduce New Data:**
    - **Terminal 2:** `echo "This is a new note from a client." > Inbox/client_notes.txt`
    - *Observe Terminal 1. The watcher will detect the file and create a new task in `Needs_Action`.*
    - **Verification:** `ls Needs_Action` to show the newly created task file.

3.  **Simulate Human Action:**
    - Open the task file in `Needs_Action` (e.g., `code Needs_Action/task_client_notes.txt...md`).
    - Manually check the box in the checklist: `- [ ] step 1` -> `- [x] step 1`, etc. for all steps.
    - Save and close the file.
    - *Explanation: "Here, we are simulating a human (or another AI) completing the steps required by the task."*

4.  **Activate the Brain:**
    - **Terminal 2:** `python process_tasks.py`
    - *Observe the output. The script will identify the completed task and move it.*
    - **Verification:**
      - `ls Needs_Action` (should be empty).
      - `ls Done` (should now contain the task file).
      - `cat Logs/System_Log.md` (should show a log entry for the processed task with status "done").

5.  **Demonstrate the Planning Skill:**
    - First, create a few more tasks by adding files to the `Inbox`. Don't complete them.
    - **Terminal 2 (Your prompt to the AI):** `Make a plan for tasks`
    - *Explanation: "I am now instructing the AI to perform a higher-level reasoning task it has learned."*
    - **AI Action:** The AI will scan `Needs_Action`, analyze the tasks, and create a `plan_...md` file in the `Plans` folder.
    - **Verification:** `ls Plans` and `cat Plans/plan_...md` to show the generated plan.

This concludes the Bronze Tier demo, showcasing a resilient, autonomous system with basic task processing and advanced planning capabilities.
